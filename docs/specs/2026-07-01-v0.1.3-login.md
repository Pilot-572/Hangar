# v0.1.3 — Login page

## Motivation
Post-launch, r/Proxmox will ask "why no login?" every hour for a week. Ship a single-account login so port 8080 stops being trusted-by-default. Defense-in-depth on top of the v0.1.1 CSRF fix.

## Model
Single account. Username + argon2id-hashed password in `hangar.yaml` under `auth:`. Flask signed-cookie session, SameSite=Lax, HttpOnly, no Secure flag (LAN is HTTP), 30-day sliding lifetime. Secret key persists to `/etc/hangar/secret_key` (0600).

## Setup flow
- Fresh install: setup wizard collects username (default admin) + password in the same form as Proxmox creds.
- Existing v0.1.2.x install: on next visit hit `/auth_setup`, one-field screen, get logged in on submit. No YAML surgery needed.

## HTMX gotcha
Polling endpoints hitting a 302 to `/login` would swap the login form into `#vms`. Guard returns 401 with `HX-Redirect: /login` header for HTMX requests; HTMX honours it as a full navigation.

## Out of scope for v0.1.3
Multiple users, RBAC, 2FA, OAuth, password strength meter, "remember me" toggle (already 30 days by default), rate limiting (argon2 verify is ~200ms which is the rate limiter), account lockout, session-device list. All rejected — for anything else use Authelia in front and `HANGAR_DISABLE_AUTH=1`.
